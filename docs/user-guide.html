MC QR Code Chip Carrier Manager

현장 오퍼레이터를 위한 설치부터 반출까지의 단계별 사용자 가이드
버전 2.1.0 · 문서 기준일 2026-05-14 · 대상: Windows 10/11 64-bit

1. 문서 목적과 대상 독자

이 문서는 처음 프로그램을 사용하는 현장 작업자가 설치부터 ATX Import, Manual Capture/OCR, QR 매칭, CSV 반출, 이력 조회까지 스스로 따라 할 수 있도록 작성된 운영 매뉴얼입니다. 프로그램 내부 구조를 몰라도 순서대로 읽고 따라 하면 일상 작업을 수행할 수 있게 구성했습니다.

이 가이드는 현장 오퍼레이터가 직접 읽고 관리하는 문서입니다. 개발 이력이나 내부 전환 정보는 사용자 작업에 필요한 범위만 포함합니다.
대상이 문서에서 봐야 할 부분목표
현장 오퍼레이터 설치, 첫 실행, ATX Mode, Manual Mode, CSV Export 측정값과 QR을 정확히 등록하고 반출 파일을 생성

이 문서의 사용 방법

  • 처음 사용하는 경우: 1장부터 9장까지 순서대로 진행합니다.
  • ATX 장비 결과만 처리하는 경우: 3장, 4장, 5장, 6장, 9장을 봅니다.
  • 수동 측정 이미지를 캡처하는 경우: 3장, 4장, 5장, 7장, 8장, 9장을 봅니다.
  • 이전 데이터를 확인하거나 백업하는 경우: 10장, 11장, 12장을 봅니다.

2. 프로그램 개요와 전체 작업 흐름

MC QR Code Chip Carrier Manager는 반도체 측정 공정에서 발생하는 ATX 장비 결과, Manual 로트카드 이미지, QR 코드, 이력 DB, CSV 반출을 하나의 데스크톱 앱으로 묶는 도구입니다. 핵심 목적은 측정값과 QR ID를 같은 슬롯에 안정적으로 연결하고, 이후 품질/서버 제출용 자료를 같은 구조로 만드는 것입니다.

  • 서버 Upload를 사용할 경우 네트워크와 로그인 정보가 준비되어 있습니다.

3.3 데이터 보존 원칙

프로그램 실행 파일은 설치 폴더에 들어가지만, 작업 데이터베이스와 캡처 이미지는 사용자 데이터 폴더에 저장됩니다. 따라서 앱을 업그레이드하거나 재설치해도 DB는 자동으로 삭제되지 않습니다.
데이터기본 위치삭제 여부
측정 이력 DB`%LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db`언인스톨해도 보존
Manual 캡처 이미지`%LOCALAPPDATA%\MCQRCodeChipCarrier\captures\`카드 삭제 후에도 파일 보존
설치 파일과 DLL`C:\Program Files\McQrManager\` 또는 사용자 선택 경로언인스톨 시 제거

4. 설치 및 첫 실행

설치 완료 기준은 단순히 설치 마법사가 끝나는 것이 아니라, 앱이 실행되고 상단 탭과 생산일자 선택 영역이 보이며, Manual/ATX 화면으로 이동할 수 있는 상태입니다.

4.1 설치 절차

  1. 설치 파일 실행

    사내 배포 경로에서 받은 최신 `McQrManager-Setup-x.y.z.exe` 파일을 더블클릭합니다.

  2. SmartScreen 경고 대응

    "Windows에서 PC를 보호했습니다" 창이 뜨면 추가 정보를 누른 뒤 실행을 선택합니다.

    코드 서명 인증서가 아직 적용되지 않은 사내 배포본에서는 SmartScreen 경고가 표시될 수 있습니다. 설치 파일 출처가 사내 배포 경로인지 먼저 확인하세요.
  3. 설치 언어와 경로 확인

    설치 언어는 한국어 또는 영어를 선택합니다. 기본 설치 경로를 그대로 사용해도 됩니다.

    관리자 권한이 없는 PC에서도 설치할 수 있도록 사용자 권한 설치가 지원됩니다.

  4. 설치 완료 후 실행

    마지막 화면에서 실행 옵션이 보이면 체크한 상태로 마침을 누릅니다. 또는 시작 메뉴에서 프로그램을 실행합니다.

  5. 첫 실행 확인

    상단에 ATX Mode, Manual Mode, CSV Export, History 탭이 보이면 정상입니다.

4.2 설치 후 정상 확인 체크리스트

  • 프로그램 제목이 `MC QR Code Chip Carrier Manager`로 표시됩니다.
  • 상단 생산일자가 오늘 날짜 또는 선택 가능한 날짜로 표시됩니다.
  • Manual Mode로 이동하면 `📸 Capture`, `Load`, `+ Add Tab`, `Refresh OCR` 버튼이 보입니다.
  • CSV Export로 이동하면 ATX / Manual 탭이 보입니다.
  • History로 이동하면 Records / Statistics 탭이 보입니다.

4.3 업그레이드와 재설치

동일한 MC 2.x 계열 설치본은 같은 AppId를 사용하므로 Windows가 같은 제품의 업데이트로 인식합니다. 설치본을 새로 실행해도 기존 작업 DB는 설치 폴더 밖에 있으므로 보존됩니다.

DB 파일을 직접 삭제하면 History와 기존 작업 이력이 사라집니다. PC 교체나 초기화 전에는 반드시 History Records에서 Export Bundle 또는 Quick Backup을 먼저 실행하세요.

4.4 실행하면 안 되는 파일

개발 또는 빌드 작업을 하는 PC에서는 `build\McQrManager\McQrManager.exe`라는 중간 산출물이 보일 수 있습니다. 이 파일은 정상 배포 실행 파일이 아니며 `python311.dll` 오류가 발생할 수 있습니다.

구분실행 여부설명
`Output\McQrManager-Setup-2.1.0.exe`실행 가능최종 설치 파일
`dist\McQrManager\McQrManager.exe`개발/검증용 실행 가능PyInstaller onedir 최종 산출물
`build\McQrManager\McQrManager.exe`실행 금지PyInstaller 중간 작업 폴더. DLL 누락 가능

5. 화면 구성과 공통 조작

5.1 상단 탭

탭주요 용도작업 완료 후 연결되는 화면

5.2 공통 입력 영역

  • 생산일자: 상단의 `생산일자:` 콤보에서 선택합니다. CSV에는 `YYYYMMDD` 형식으로 저장됩니다.
  • QR 입력창: 화면 하단에 있습니다. 카드 선택 후 QR 스캐너를 찍으면 선택 슬롯에 QR이 매칭됩니다.
  • 진행률: 하단 또는 상단에 `0 / 12 matched` 같은 형식으로 QR 매칭 상태가 표시됩니다.
  • Log: ATX, Manual, CSV Export 화면 하단에서 처리 결과와 경고를 확인합니다. History 화면은 상세 조회 공간 확보를 위해 전용 Log 박스가 없습니다.

5.3 카드 선택과 QR 매칭

  1. 카드 선택

    ATX 또는 Manual 화면에서 작업할 카드를 클릭합니다. 선택된 카드는 테두리 또는 강조 색으로 표시됩니다.

  2. QR 입력창 확인

    하단 QR 입력창의 placeholder가 선택 카드 번호와 Probe Type을 보여주는지 확인합니다.

  3. QR 스캔

    QR 스캐너로 코드를 찍습니다. 스캐너는 일반 키보드 입력처럼 QR 문자열을 입력하고 Enter를 보내는 방식이어야 합니다.

  4. 매칭 결과 확인

    카드 상태가 QR 완료 상태로 바뀌고, 진행률 숫자가 증가하면 정상입니다.

이미 다른 슬롯에 사용된 QR ID를 입력하면 중복 방지를 위해 차단됩니다. 중복 경고가 뜨면 실제 제품 QR 또는 선택 카드가 맞는지 확인하세요.

6. ATX Mode 사용 가이드

ATX Mode는 장비가 이미 생성한 `summary_*.csv` 파일을 로트 단위로 불러와 슬롯별 Frequency/Q/Probe 정보를 자동 등록하는 화면입니다.

6.1 사용 시점과 완료 기준

항목기준
사용 시점ATX 장비에서 측정 결과 폴더가 생성된 후
입력 데이터`summary_1.csv`, `summary_2.csv` 등 슬롯별 summary CSV
완료 기준모든 슬롯 측정값 확인, 필요한 QR 매칭, CSV Export 가능 상태

6.2 ATX 폴더 Import

  1. ATX Mode 선택

    상단 탭에서 ATX Mode를 클릭합니다.

  2. 장비 결과 폴더 선택

    Import 또는 Browse 버튼으로 ATX 장비가 저장한 폴더를 선택합니다. 폴더명은 PO Number와 Probe 정보를 포함하는 것이 좋습니다.

    예: `P2601002_12M_AC160`

  3. summary CSV 확인

    선택한 폴더 안에 `summary_*.csv` 파일이 있어야 합니다. 파일이 없으면 Import가 실패합니다.

  4. 슬롯 카드 확인

    Import가 끝나면 우측에 슬롯 카드가 생성됩니다. 각 카드에서 Frequency, Q, QR 상태를 확인합니다.

  5. QR 매칭

    슬롯 카드를 클릭한 뒤 QR 스캐너로 해당 제품 QR을 입력합니다. 다음 슬롯으로 이동해 반복합니다.

6.3 슬롯 수정

측정값, Probe Type, QR ID, Source를 수정해야 할 때는 카드 우클릭 메뉴를 사용합니다.

  1. 수정할 카드 우클릭

    카드에서 마우스 오른쪽 버튼을 누르고 수정...을 선택합니다.

  2. 다이얼로그에서 값 수정

    Probe Type, Frequency, Q-factor, QR ID, Source 값을 확인하고 필요한 항목만 바꿉니다.

  3. OK 또는 Cancel

    OK를 누르면 중복 QR 검사를 통과한 뒤 저장됩니다. Cancel을 누르면 원본 슬롯은 바뀌지 않습니다.

6.4 ATX 작업 완료 확인

  • 선택한 폴더의 PO Number와 Probe Type이 예상과 일치합니다.
  • 슬롯 수가 장비 결과와 일치합니다.
  • Frequency/Q 값이 빈 칸이거나 비정상인 슬롯을 확인했습니다.
  • 필요한 QR ID를 모두 입력했습니다.
  • CSV Export의 ATX 탭에서 동일 데이터가 보입니다.

6.5 ATX Mode 자주 발생하는 문제

증상가능한 원인조치
summary CSV를 찾을 수 없음잘못된 폴더 선택탐색기에서 `summary_*.csv`가 있는 폴더를 직접 확인

QR 중복 경고같은 QR을 다른 슬롯에 이미 입력History/카드 상태를 확인하고 실제 QR을 다시 스캔

7. Manual Mode Capture/OCR 사용 가이드

Manual Mode의 표준 공정은 Probe Type 선택 → Zoom-In 캡처 → OCR 확인 → QR 스캔입니다. 파일명을 외우지 않아도, 방금 캡처한 카드에 바로 QR을 찍는 흐름으로 설계되어 있습니다.

7.1 Manual 작업 준비

  • Manual Mode 탭으로 이동했습니다.
  • 작업할 Probe Type 탭을 선택했습니다. Overview 탭에서는 캡처를 시작하지 않습니다.
  • Sweep 이미지 또는 측정 팝업 창이 화면에 열려 있습니다.
  • QR 스캐너가 준비되어 있습니다.

7.2 Capture 메뉴와 단축키

작업단축키사용 시점결과
Zoom-In · Region CaptureF6숫자 영역을 직접 드래그할 때새 Manual 카드 생성
Zoom-In · Window CaptureF7Sweep 팝업 창 전체를 일정 크기로 캡처할 때새 Manual 카드 생성
Zoom-Out · Region CaptureF8선택 카드의 전체 화면 문맥 이미지를 추가할 때선택 카드의 Zoom-Out 이미지 생성
Zoom-Out · Window CaptureF9선택 카드의 전체 창 이미지를 추가할 때선택 카드의 Zoom-Out 이미지 생성
Zoom-Out 캡처는 새 카드를 만들지 않습니다. 이미 선택된 카드에 보조 이미지를 붙이는 작업입니다. 먼저 Zoom-In 캡처로 카드가 생성되어 있어야 합니다.

7.3 Zoom-In 캡처 후 OCR/QR 작업

  1. Probe Type 탭 선택

    현재 작업할 Probe Type 탭을 클릭합니다. 탭이 없으면 `+ Add Tab`으로 추가합니다.

  2. 캡처 시작

    📸 Capture 메뉴에서 `Zoom-In · Region Capture` 또는 `Zoom-In · Window Capture`를 선택합니다. 단축키를 사용하려면 F6 또는 F7을 누릅니다.

  3. 영역 또는 창 선택

    Region Capture는 마우스로 영역을 드래그합니다. Window Capture는 캡처할 Sweep 창을 클릭합니다. 취소하려면 Esc 또는 우클릭을 사용합니다.

  4. 카드 자동 생성 확인

    캡처가 저장되면 우측에 새 카드가 생성되고 자동 선택됩니다. 좌측 Sweep Image에 방금 캡처한 이미지가 표시됩니다.

  5. OCR 결과 확인

    Frequency (KHz), Q 입력칸에 숫자가 자동 입력되었는지 확인합니다. 틀리면 직접 수정하고 Apply를 누릅니다.

  6. QR 스캔

    하단 QR 입력창에 포커스가 자동 이동합니다. QR 스캐너를 찍으면 선택 카드에 QR이 매칭됩니다.

7.4 Zoom-Out 캡처

Zoom-Out 이미지는 같은 슬롯의 전체 문맥을 보존하는 용도입니다. CSV+Images Export 시 `ZOOMOUT` 폴더에 같이 반출됩니다.

  1. Zoom-In 카드 선택

    Zoom-Out 이미지를 붙일 Manual 카드를 클릭합니다.

  2. Zoom-Out 캡처 실행

    캡처 메뉴에서 `Zoom-Out · Region Capture` 또는 `Zoom-Out · Window Capture`를 선택합니다. 단축키는 F8, F9입니다.

  3. 이미지 뷰어 전환 확인

    좌측 이미지 뷰어 상단의 Zoom-In / Zoom-Out 버튼으로 두 이미지를 전환해 확인합니다.

7.5 기존 이미지 Load / Drag & Drop

캡처 기능을 쓰지 않고 기존 이미지 파일을 불러올 수도 있습니다. `Load` 버튼을 누르거나 이미지 파일을 Manual 화면으로 드래그 앤 드롭합니다. 이 방식으로 불러온 외부 원본 이미지는 QR 매칭 후에도 자동 rename하지 않습니다.

7.6 잘못 캡처한 카드 삭제

잘못 캡처한 카드를 삭제하면 남은 Manual 카드는 현재 화면 순서 기준으로 `#1`, `#2`, `#3` 순서로 다시 정렬됩니다. QR ID, Frequency, Q, Probe Type, 이미지 경로는 해당 카드와 함께 유지됩니다.

앱 내부 캡처 파일은 순번 재정렬에 맞춰 파일명이 동기화될 수 있습니다. 외부에서 Load한 원본 이미지는 파일명을 바꾸지 않습니다.

7.7 Manual 작업 완료 확인

  • Probe Type 탭이 올바릅니다.
  • 카드 번호가 현재 작업 순서와 일치합니다.
  • Frequency와 Q 값이 실제 화면과 일치합니다.
  • 필요한 카드에 QR ID가 매칭되었습니다.
  • Zoom-Out 이미지가 필요한 슬롯은 Zoom-Out 탭에서 확인됩니다.
  • CSV Export의 Manual 탭에서 동일 데이터가 보입니다.

8. OCR ROI Calibrator 사용 가이드

OCR이 실패하거나 다른 해상도의 이미지가 들어왔을 때만 ROI Calibrator를 사용합니다. 정상적으로 숫자가 읽히는 해상도에서는 매번 조정할 필요가 없습니다.

8.1 Calibrator가 필요한 상황

  • Frequency 또는 Q가 항상 빈 칸으로 남습니다.
  • 로그에 "저장된 ROI 프로파일 없음" 또는 해상도 관련 안내가 표시됩니다.
  • 캡처 창 크기나 Windows 배율이 바뀐 뒤 OCR이 갑자기 실패합니다.
  • 새 장비 화면 또는 새 레이아웃의 Sweep 이미지를 사용합니다.

8.2 ROI 의미

항목의미맞춰야 하는 영역
Frequency ROIFrequency (KHz) 숫자를 읽는 박스Frequency 값 숫자만 포함
Q Factor ROIQ 값 숫자를 읽는 박스Q 값 숫자만 포함
x, y이미지 왼쪽 위 기준 위치숫자 박스의 시작점
w, hROI 너비와 높이숫자가 잘리지 않고 들어가는 크기

8.3 Calibrator 사용 절차

  1. Calibrate OCR 열기

    상단 🎯 Calibrate OCR 버튼을 누릅니다. History 탭에서는 이 버튼이 숨겨져 있으므로 ATX, Manual, CSV Export 화면에서 실행합니다.

  2. 기준 이미지 로드

    `Load reference image...`를 눌러 실제 작업에 사용할 이미지와 같은 해상도의 샘플 이미지를 선택합니다.

  3. ROI 박스 조정

    Frequency ROI와 Q Factor ROI가 각각 숫자 부분만 포함하도록 x, y, w, h 값을 조정합니다.

  4. Test OCR 실행

    `Run OCR on current ROI`를 눌러 Frequency와 Q가 읽히는지 확인합니다.

  5. Save 또는 Cancel

    정상적으로 읽히면 Save를 누릅니다. Cancel을 누르면 변경한 ROI는 저장되지 않습니다.

8.4 ROI 조정 팁

  • 숫자 주변의 라벨 텍스트까지 포함하면 OCR 오탐이 늘어날 수 있습니다.
  • 숫자가 너무 작게 들어가면 w/h를 약간 키웁니다.
  • 다른 해상도 이미지는 별도 프로파일로 저장됩니다.
  • Windows 배율 125%/150%에서는 캡처 이미지 크기가 달라질 수 있으므로 실제 운영 배율에서 보정합니다.

9. CSV Export / Upload / CSV+Images 사용 가이드

CSV Export 화면의 모든 작업은 현재 활성 탭 기준입니다. ATX 데이터를 반출하려면 ATX 탭을, Manual 데이터를 반출하려면 Manual 탭을 선택한 뒤 Save 또는 Upload를 실행합니다.

9.1 CSV Export 화면 구성

  1. CSV Export 탭 선택

    상단에서 CSV Export를 클릭합니다.

  2. ATX / Manual 탭 선택

    우측 테이블 위의 ATX 또는 Manual 탭을 선택합니다. 진행률, 미리보기, Save/Upload는 이 활성 탭 기준으로 동작합니다.

  3. 슬롯 테이블 확인

    Probe Type, Freq, Q, QR ID, Status 컬럼을 확인합니다. QR이 없는 슬롯은 상태가 미완성으로 표시될 수 있습니다.

9.2 Save CSV

CSV 파일만 필요한 경우 Save CSV를 사용합니다. 저장 위치와 파일명을 선택하면 CSV가 생성됩니다.

9.3 CSV+Images

CSV와 이미지 폴더를 함께 제출해야 하는 경우 Save CSV 메뉴에서 CSV+Images 작업을 선택합니다. 생성 폴더명 기본값은 원본 폴더명 또는 이미지 공통 부모 폴더명을 우선 사용합니다. 

P2601002_12M_AC160/
├── P2601002_12M_AC160_QR.csv
├── ZOOMIN/
│   ├── QR123456.png
│   └── slot_02.png
└── ZOOMOUT/
    ├── QR123456.png
    └── slot_02.png
QR ID가 없는 슬롯 이미지는 `slot_01.png` 같은 슬롯명 기반으로 저장됩니다. 같은 이름이 이미 있으면 `_1`, `_2` 접미사가 붙습니다.

9.4 미완성 데이터 옵션

QR ID가 없는 슬롯이 있으면 경고창이 뜨고 반출 정책을 선택할 수 있습니다.

옵션결과사용 상황
QR 있는 값만 반출/업로드QR ID가 있는 슬롯만 CSV 행으로 생성서버가 빈 QR을 허용하지 않을 때
전체 슬롯 반출/업로드현재까지 입력된 모든 슬롯을 순서대로 CSV 행으로 생성측정값 자체를 먼저 보존해야 할 때
취소저장 또는 업로드를 중단누락 QR을 먼저 보완할 때

9.5 CSV 컬럼

순서컬럼명값 규칙
1QR IDQR이 없으면 빈 칸
2생산일자[YYYYMMDD]상단 생산일자 기준
3Frequency (KHz)값이 없으면 빈 칸
4Drive (%)현재 GUI 입력 항목은 아니며 값이 없으면 빈 칸
5Q값이 없으면 빈 칸
6Probe Type슬롯 값 우선, 없으면 세트 기본값

9.6 Upload

Upload는 서버 제출용 기능입니다. 서버 연결이 필요하며, 업로드가 끝나면 History의 Upload 상태가 갱신됩니다. 업로드 중 탭을 바꿔도 시작 시점의 데이터 기준으로 상태가 기록됩니다.

9.7 Export 완료 확인

  • 활성 탭이 반출하려는 모드(ATX 또는 Manual)와 일치합니다.
  • CSV 헤더가 `QR ID`, `생산일자[YYYYMMDD]`, `Frequency (KHz)`, `Drive (%)`, `Q`, `Probe Type` 순서입니다.
  • 미완성 데이터 옵션을 현장 기준에 맞게 선택했습니다.
  • CSV+Images인 경우 `ZOOMIN`과 `ZOOMOUT` 폴더가 생성되었습니다.
  • Upload 작업이면 History에서 Upload 상태를 확인했습니다.

    이전 PC에서 만든 ZIP 파일을 선택합니다.

  4. 중복 정책 확인

    신규 PC라면 Overwrite 또는 기본 정책으로 복원합니다. 기존 데이터가 있는 PC에서는 중복 정책을 신중히 선택합니다.

  5. 복원 결과 확인

    History 목록과 상세 이미지가 정상인지 확인합니다.

Bundle ZIP은 내부에 JSONL과 이미지 파일이 들어 있지만, 직접 편집하지 마세요. 수동 편집하면 Import가 실패하거나 데이터 정합성이 깨질 수 있습니다.

15. FAQ

Q. 인터넷이 없어도 사용할 수 있나요?

네. ATX Import, Manual Capture/OCR, QR 매칭, History, CSV 저장은 오프라인에서 동작합니다. 서버 Upload만 네트워크가 필요합니다.

Q. 앱을 업그레이드하면 기존 데이터가 사라지나요?

아니오. DB는 설치 폴더가 아니라 `%LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db`에 저장되므로 일반 업그레이드로 삭제되지 않습니다.

Q. 여러 PC가 같은 DB를 동시에 공유할 수 있나요?

현재는 1 PC = 1 로컬 DB 구조입니다. 여러 PC 간 데이터 이동은 Export Bundle / Import Bundle로 처리합니다.

Q. OCR이 한글도 읽나요?

현재 OCR은 Frequency와 Q 같은 숫자 추출에 맞춰져 있습니다. 한국어 문장 인식은 현재 운영 범위가 아닙니다.

Q. QR 없는 슬롯도 CSV에 남길 수 있나요?

네. 미완성 데이터 경고창에서 전체 슬롯 반출/업로드를 선택하면 QR이 없는 슬롯도 빈 QR 칸으로 CSV에 포함됩니다.

Q. Manual 카드 삭제 시 실제 이미지 파일도 삭제되나요?

아니오. 현재는 DB/카드 삭제만 수행하고 캡처 원본 파일은 보존합니다. 파일 정리 정책은 별도 운영 기준이 필요합니다.
