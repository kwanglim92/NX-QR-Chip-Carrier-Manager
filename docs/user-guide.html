MC QR Code Chip Carrier Manager

칩 캐리어 측정 이력 관리 — 사용자 가이드
v2.0 · 2026-04-24 · 공장 측정 오퍼레이터·품질 관리자 대상

1. 제품 개요

MC QR Code Chip Carrier Manager 는 반도체 측정 공장에서 칩 캐리어의 측정 이력을 체계적으로 관리하는 Windows 데스크톱 프로그램입니다.

이 프로그램이 해결해 주는 문제

기존 방식의 문제이 프로그램의 해결
ATX 장비가 슬롯마다 summary_*.csv 를 쏟아내 한 로트의 전체 상태 파악이 어려움 폴더 하나만 지정하면 12슬롯이 한 화면에 자동 정렬
로트카드 Manual 측정 값을 눈으로 읽어 수기 입력 → 오탈자 빈발 이미지 드래그 한 번에 OCR 로 Frequency / Q-factor 자동 입력
이력이 각 PC 의 Excel 파일에 흩어져 일일 생산량 집계가 번거로움 SQLite DB 에 자동 저장 → Dashboard 로 즉시 집계
다른 PC 로 데이터를 이전하려면 Excel 을 수작업으로 합쳐야 함 Export 번들(ZIP) 로 한 파일에 묶어 USB 전송 → Import 로 병합

누가 쓰는가

  • 공장 측정 오퍼레이터 — ATX 결과 Import, Manual 로트카드 OCR, 저장
  • 품질 관리자 — Dashboard 생산량 모니터링, Export 번들 배포
  • IT 담당자 — 인스톨러 배포, PC 간 데이터 이전

언제 쓰는가

  • ATX 장비로 측정 직후 → 폴더 Import 로 12슬롯 일괄 등록
  • 로트카드로 Manual 측정 시 → 이미지 드롭으로 OCR 자동 입력
  • 일간·주간 보고서 작성 전 → Dashboard 로 통계 확인
  • PC 교체·백업·본사 취합 → Export 번들 생성·공유
모든 기능은 오프라인에서도 동작 합니다. 네트워크가 필요한 것은 "서버 업로드" 기능 하나뿐입니다.

2. 설치

시스템 요구사항

  • Windows 10 / 11 (64-bit)
  • 여유 공간 약 600 MB (Tesseract OCR 엔진 포함)
  • 관리자 권한 불필요 (사용자 권한으로 설치)
  • 인터넷 연결 불필요 (서버 업로드 기능 제외)

설치 단계

  1. 설치 파일 실행

    IT 담당자에게 받은 McQrManager-Setup-2.0.0.exe 파일을 더블클릭합니다.

  2. SmartScreen 경고 대응 (처음 설치 시)

    "Windows에서 PC를 보호했습니다" 창이 뜨면 "추가 정보" → "실행" 순서로 클릭합니다. (코드 서명 인증서 도입 전까지 표시되는 정상적인 메시지입니다.)

  3. 언어 선택

    "한국어" 또는 "English" 중 선택 후 확인.

  4. 설치 경로 선택

    기본값 C:\Program Files\McQrManager\ 권장. 특별한 이유가 없다면 그대로 다음.

  5. 추가 작업 선택

    바탕화면 아이콘 만들기 체크박스 — 필요 시 선택. 기본은 해제 상태.

  6. 설치 진행

    설치 클릭 → 1~2 분 대기. 진행률 바가 100% 에 도달하면 완료.

  7. 완료

    MC QR Code Chip Carrier Manager 실행 체크박스가 선택된 상태로 마침 클릭 → 첫 실행으로 이어집니다.

이 프로그램은 VC++ Redistributable 등 별도 설치 불필요합니다. Tesseract OCR 엔진까지 모두 포함되어 있습니다.

3. 첫 실행

  1. 로그인 다이얼로그

    사용자 이름을 입력하고 확인. 이 이름은 이후 모든 측정 기록에 "작업자"로 저장됩니다.

  2. DB 자동 생성

    최초 실행 시 %LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db 가 자동으로 생성됩니다. 사용자가 따로 할 일은 없습니다.

  3. 메인 윈도우 — 4개 탭 투어

    상단의 4개 탭을 한 번씩 눌러보세요.

    • ATX — 측정 장비의 summary CSV 폴더를 Import 하는 화면
    • Manual — 로트카드 이미지를 드래그 앤 드롭하여 OCR 로 값 입력
    • History — 과거 측정 이력을 필터·검색
    • Dashboard — Today 카드 + 기간별 차트

4. ATX 모드 — 측정 장비 결과 자동 Import

ATX 장비가 생성한 summary_*.csv 파일이 들어있는 폴더를 지정하면 12슬롯을 한 번에 등록합니다.

  1. ATX 탭 진입 → "Import ATX Folder" 버튼 클릭
  2. 폴더 선택

    측정 장비가 결과를 저장한 폴더를 선택합니다. 안에 summary_1.csv ~ summary_12.csv 같은 파일들이 있어야 합니다.

  3. 메타 입력

    좌측 입력 패널에서:

    • PO 번호 (필수) — 로트 식별자
    • 주문 수량
    • 프로브 유형 (드롭다운)
    • 생산 일자 (기본: 오늘)
  4. QR 스캐너로 슬롯별 QR 등록

    각 슬롯 카드에 QR 코드 입력 칸이 있습니다. USB QR 스캐너로 순서대로 스캔하면 자동으로 다음 슬롯으로 이동합니다.

  5. 저장 또는 업로드
    • Save — 로컬 DB 저장 (오프라인 OK)
    • Upload — 서버 전송 (네트워크 필요)

    둘 다 History 탭에 즉시 반영됩니다.

자주 발생하는 트러블

증상원인 / 해결
"summary CSV 를 찾을 수 없음" 잘못된 폴더 선택. summary_1.csv 등이 있는지 탐색기로 직접 확인.
슬롯 수가 12개가 아님 (예: 11개) 일부 summary 파일 누락. 장비에서 해당 슬롯 재측정.
QR 스캐너로 스캔했는데 반응 없음 QR 입력 칸에 포커스가 있는지 확인. 첫 칸을 마우스로 클릭 후 스캔.

5. Manual 모드 — 로트카드 OCR 자동 입력

로트카드 사진/스캔 이미지를 드래그 앤 드롭하면 Frequency / Q-factor 값을 자동으로 읽어옵니다.

  1. Manual 탭 진입
  2. 로트카드 이미지 드래그 앤 드롭

    탐색기에서 JPG / PNG 파일을 중앙의 드롭 영역으로 끌어다 놓습니다. 여러 장을 한 번에 드롭해도 됩니다 (배치 처리).

    여러 해상도 이미지를 한꺼번에 드롭해도 OK. 각 이미지의 해상도에 맞는 ROI 프로파일이 자동으로 선택됩니다.
  3. OCR 자동 실행 (비동기)

    배경 스레드 4개가 동시에 OCR 을 수행합니다. 20장 드롭해도 UI 가 멈추지 않습니다. 진행 중엔 각 카드 하단에 스피너가 돕니다.

  4. 결과 확인 + 교정

    OCR 이 끝나면 Frequency / Q-factor 칸에 숫자가 자동 입력됩니다.

    • 값이 맞으면 그대로 두세요.
    • 값이 틀렸으면 직접 수정 가능합니다.
    • OCR 이 실패한 카드는 빈 칸으로 남습니다 (경고 없음) → 수기 입력.
  5. QR 스캐너로 각 카드에 QR 등록

    ATX 모드와 동일하게 순서대로 스캔.

  6. Save → History 에 반영

5.1 ROI Calibrator (해상도 다른 로트카드 추가 시)

기본 제공 프로파일에 없는 새 해상도의 로트카드를 사용할 때, OCR 이 제대로 되지 않으면 ROI(관심 영역)를 직접 설정합니다.

  1. 메뉴 → Tools → ROI Calibrator 열기
  2. 샘플 이미지 로드

    Load Image… 버튼으로 문제가 된 해상도의 로트카드 한 장을 불러옵니다.

  3. ROI 박스를 드래그해서 숫자 영역으로 이동

    파란 박스는 Frequency, 빨간 박스는 Q-factor. 각각 숫자가 표시되는 정확한 사각형 영역으로 이동·리사이즈.

  4. Test OCR 버튼으로 확인

    추출된 값이 카드에 적힌 숫자와 일치하는지 확인.

  5. Save Profile 클릭

    해당 해상도 기준으로 ROI 가 DB(`app_settings.ocr_roi`)에 저장됩니다. 다음부터 같은 해상도 이미지를 드롭하면 자동 적용.

OCR 정확도는 이미지 품질에 크게 좌우됩니다. 초점이 흐리거나 조명이 너무 어두우면 실패할 수 있습니다. 카메라·스캐너 설정을 균일하게 유지하세요.

6. History — 측정 이력 조회

  1. History 탭 진입
  2. 필터 설정
    • 날짜 범위 — 캘린더 위젯으로 시작/종료일 선택
    • PO 번호 — 부분 일치 검색
    • Probe Type — 드롭다운
    • Upload Status — pending / uploaded / failed
    • QR 코드 — 특정 QR 로 검색
  3. 정렬

    컬럼 헤더 클릭으로 정렬. 기본은 최신순 (production_date DESC).

  4. 행 더블클릭 → 슬롯 상세

    측정 세트를 더블클릭하면 12슬롯의 QR / Freq / Drive / Q 값이 테이블 형태로 표시됩니다.

  5. 업로드 실패 건 재전송

    "Upload Status = failed" 로 필터 → 행 선택 → 우클릭 → Retry Upload.

7. Dashboard — 생산량 대시보드

7.1 상단 — Today 카드

  • Today Sets — 오늘 측정한 세트(로트) 수
  • Today Slots — 오늘 측정한 슬롯 총 개수
  • Today Complete % — QR 입력 완료율
  • Today Pending — 업로드 대기 세트 수

7.2 기간 스위치

대시보드 상단의 토글 버튼:

기간집계 단위
Daily일자별 (YYYY-MM-DD)
WeeklyISO 주차 (YYYY-W##)
Monthly월별 (YYYY-MM)
Quarterly분기 (YYYY-Q#)
Yearly연도

7.3 차트 4종 해석

  1. 시계열 바 차트 (좌상) — 선택한 기간의 세트 수 추이
  2. 업로드 상태 파이 (우상) — pending/uploaded/failed 비율
  3. 프로브 유형별 누적 바 (좌하) — Probe Type 별 슬롯 합계
  4. 주간 히트맵 (우하) — 요일 × 시간대 생산 분포
차트는 필터 조건이 History 탭과 연동됩니다. History 에서 특정 PO 로 필터한 뒤 Dashboard 로 이동하면 해당 PO 데이터만 집계됩니다.

8. Export / Import — 번들로 데이터 이전

측정 이력을 ZIP 파일 한 개로 묶어 다른 PC 에 전송할 수 있습니다.

8.1 Export — 번들 생성

  1. 메뉴 → File → Export Bundle
  2. 필터 지정
    • 전체 데이터 / 특정 기간 / 특정 PO 선택
    • 이미지 포함 체크박스 — Manual 로트카드 이미지도 함께 내보낼지
  3. 저장 경로 선택

    기본 파일명: chip_carrier_export_YYYYMMDD.zip

  4. Export 클릭 → 완료 메시지

8.2 Import — 번들 불러오기

  1. 메뉴 → File → Import Bundle
  2. ZIP 파일 선택
  3. 미리보기 확인

    스키마 버전 / 레코드 수 / 날짜 범위 / 이미지 포함 여부가 표시됩니다.

  4. 중복 정책 선택
    정책동작
    Skip (기본)같은 QR 이 이미 있으면 건너뜀. 새 데이터만 추가.
    Overwrite같은 QR 이 있으면 기존 레코드를 번들 데이터로 덮어쓰기.
    Merge세트는 유지하고 누락된 슬롯만 번들에서 채움.
  5. Import 실행

    진행률 표시 후 완료 메시지. History 에 즉시 반영.

8.3 사용 시나리오

  • 백업 — 주 1회 전체 Export → NAS/USB 보관
  • PC 교체 — 구 PC 에서 Export → 신 PC 에서 Import (Overwrite)
  • 본사 취합 — 각 공장 PC 에서 Export → 본사 PC 에서 Merge 로 통합
Export 번들은 JSONL + ZIP 으로 열어볼 수 있으나 수동 편집하지 마세요. 스키마 검증에 실패할 수 있습니다. 상세 스키마는 docs/export_schema.md 참조.

9. 설정

메뉴 → Edit → Settings (또는 Ctrl+,)

주요 항목

  • 사용자 이름 — 로그인 시 입력한 이름 변경
  • 서버 업로드 URL — 회사 내부 서버 엔드포인트
  • OCR ROI 프로파일 — Calibrate… 버튼으로 Calibrator 진입 (§5.1 참조)
  • 로그 레벨 — All / Warnings+ / Errors only
  • DB 경로 (읽기 전용) — %LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db
설정은 DB 의 app_settings 테이블에 저장되므로 앱 재시작 후에도 유지됩니다. 업그레이드해도 보존됩니다.

10. 트러블슈팅

Q1. OCR 이 전부 실패해요

  1. 이미지 해상도가 기본 프로파일과 다른지 확인 → ROI Calibrator 에서 신규 프로파일 추가 (§5.1)
  2. 이미지가 흐릿하거나 너무 어두운지 확인 → 카메라 조명·초점 조정
  3. OCR 결과가 항상 None 이면 Tesseract 가 번들에 포함되지 않았을 수 있음 → IT 담당자에게 재설치 요청

Q2. 한글 폴더나 한글 사용자 계정에서 오류가 나요

이 프로그램은 Windows 8.3 단축 경로(short path) 를 자동 변환해 Tesseract 호환 문제를 회피합니다. 재현되면 로그를 첨부해 개발팀에 보고하세요.

Q3. 서버 업로드가 실패해요

  1. Settings 에서 서버 URL 이 올바른지 확인
  2. 네트워크 연결 상태 확인 (사내망 / VPN)
  3. History 에서 "failed" 로 필터 → 우클릭 → Retry Upload

Q4. 앱이 아예 켜지지 않아요

DB 파일 손상 가능성. 데이터가 손실될 수 있으니 신중히:

  1. %LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db 를 다른 이름으로 백업
  2. 원본 DB 삭제
  3. 앱 재실행 (신규 DB 자동 생성)
  4. 백업한 DB 를 SQLite 도구로 열어 복구 가능한 데이터 확인
DB 파일을 삭제하면 모든 이력이 사라집니다. 정기적으로 Export 번들로 백업하세요 (§8).

Q5. SmartScreen 경고가 뜹니다

현재 코드 서명 인증서가 없어 Windows 가 "알 수 없는 게시자" 로 표시합니다. "추가 정보" → "실행" 로 진행하세요. 정상적인 동작입니다.

Q6. 이전 버전(NX 1.0.0) 데이터를 불러오고 싶어요

§12 (부록 — 1.0.0 → 2.0.0 마이그레이션) 을 참조하세요.

로그 파일 위치

현재는 UI 로그창에만 표시됩니다 (ATX/Manual/Export/History 탭 하단). 스크린샷을 찍어 지원팀에 보내주세요.

11. FAQ

Q. 여러 사람이 같은 DB 를 공유할 수 있나요?

아니오. 현재는 1 PC = 1 DB 구조입니다. 여러 PC 간 데이터 공유는 Export/Import 번들로 해결합니다.

Q. 데이터는 어디에 저장되나요?

%LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db — 탐색기 주소창에 %LOCALAPPDATA%\MCQRCodeChipCarrier 를 붙여 넣으면 바로 이동합니다.

Q. 앱을 업그레이드하면 기존 데이터가 지워지나요?

아니오. DB 는 설치 경로 외부에 있어 업그레이드·재설치 시 보존됩니다.

Q. 인터넷 없어도 쓸 수 있나요?

네. 서버 업로드 기능을 제외한 모든 기능이 오프라인에서 동작합니다.

Q. OCR 이 한국어를 인식하나요?

현재는 영문 + 숫자 전용입니다. 로트카드의 숫자 값 추출에 최적화되어 있습니다. 한국어 OCR 은 차기 버전 검토 항목입니다.

Q. 이 프로그램의 라이선스는?

사내 전용 프로그램입니다. 포함된 Tesseract 는 Apache License 2.0 입니다 (`third_party/tesseract/LICENSE.txt` 참조).

12. 부록 — 1.0.0 (NX) → 2.0.0 (MC) 마이그레이션

이전 버전 NX QR Chip Carrier Manager 1.0.0 사용자가 MC 2.0.0 으로 데이터를 이전하는 절차입니다.

두 버전은 AppId 가 달라 Windows 는 별도 제품으로 인식합니다. 업그레이드 설치가 아니라 신규 설치이며, DB 경로도 변경되므로 자동 이관되지 않습니다.

절차

  1. NX 1.0.0 에서 전체 Export
    • History 탭 → 전체 기간 선택
    • 메뉴 → File → Export Bundle → 이미지 포함 체크
    • 저장 경로: USB / 네트워크 드라이브 / 바탕화면
  2. (선택) NX DB 백업

    %LOCALAPPDATA%\NXQRChipCarrierManager\ 폴더 전체를 복사하여 별도 보관. 만약 번들에 누락이 있어도 SQLite 도구로 복구 가능.

  3. MC 2.0.0 설치

    §2 절차대로 설치. NX 1.0.0 은 유지해도 되고 언인스톨해도 됩니다.

  4. MC 2.0.0 에서 Import
    • 메뉴 → File → Import Bundle
    • 1번에서 만든 ZIP 선택
    • 중복 정책: Overwrite (신규 설치라 충돌 없음)
    • Import 클릭 → History 에 모든 데이터 복원 확인
  5. (선택) NX 1.0.0 언인스톨

    제어판 → 앱 및 기능 → NX QR Chip Carrier Manager 제거. DB 폴더(%LOCALAPPDATA%\NXQRChipCarrierManager\)는 수동으로 삭제해야 완전히 정리됩니다.

문제가 생겨도 NX 1.0.0 DB 와 번들 ZIP 이 둘 다 있으면 언제든 복구 가능합니다. 마이그레이션 완료 후 충분한 기간(예: 1개월) 작업 이상 없음을 확인한 뒤 NX 1.0.0 과 구 DB 를 정리하세요.
