MC QR Code Chip Carrier Manager

칩 캐리어 측정 이력 관리 — 사용자 가이드
v2.1 · 2026-05-14 · 공장 측정 오퍼레이터·품질 관리자 대상

1. 제품 개요

MC QR Code Chip Carrier Manager 는 반도체 측정 공장에서 칩 캐리어의 측정 이력을 체계적으로 관리하는 Windows 데스크톱 프로그램입니다.

이 프로그램이 해결해 주는 문제

기존 방식의 문제이 프로그램의 해결
ATX 장비가 슬롯마다 summary_*.csv 를 쏟아내 한 로트의 전체 상태 파악이 어려움 폴더 하나만 지정하면 12슬롯이 한 화면에 자동 정렬
로트카드 Manual 측정 값을 눈으로 읽어 수기 입력하고, Windows 캡처 도구로 이미지를 따로 저장해야 함 앱 안에서 Sweep 화면을 바로 Capture → OCR → QR 스캔 흐름으로 처리
이력이 각 PC 의 Excel 파일에 흩어져 일일 생산량 집계가 번거로움 SQLite DB 에 자동 저장 → History Statistics 로 즉시 집계
다른 PC 로 데이터를 이전하려면 Excel 을 수작업으로 합쳐야 함 CSV/CSV+Images 또는 Export Bundle 로 목적에 맞게 반출·백업

누가 쓰는가

  • 공장 측정 오퍼레이터 — ATX 결과 Import, Manual Capture/OCR, QR 매칭
  • 품질 관리자 — History Statistics 생산량 모니터링, CSV Export 검토
  • IT 담당자 — 인스톨러 배포, PC 간 데이터 이전

언제 쓰는가

  • ATX 장비로 측정 직후 → 폴더 Import 로 12슬롯 일괄 등록
  • 로트카드로 Manual 측정 시 → Region/Window Capture 로 이미지 등록 후 OCR 자동 입력
  • 일간·주간 보고서 작성 전 → History Statistics 로 통계 확인
  • 고객/서버 제출 → CSV 또는 CSV+Images Export
  • PC 교체·백업·본사 취합 → Export Bundle 생성·공유
모든 기능은 오프라인에서도 동작 합니다. 네트워크가 필요한 것은 "서버 업로드" 기능 하나뿐입니다.

2. 설치

시스템 요구사항

  • Windows 10 / 11 (64-bit)
  • 여유 공간 약 600 MB (Tesseract OCR 엔진 포함)
  • 관리자 권한 불필요 (사용자 권한으로 설치)
  • 인터넷 연결 불필요 (서버 업로드 기능 제외)

설치 단계

  1. 설치 파일 실행

    IT 담당자에게 받은 최신 McQrManager-Setup-x.y.z.exe 파일을 더블클릭합니다.

  2. SmartScreen 경고 대응 (처음 설치 시)

    "Windows에서 PC를 보호했습니다" 창이 뜨면 "추가 정보" → "실행" 순서로 클릭합니다. (코드 서명 인증서 도입 전까지 표시되는 정상적인 메시지입니다.)

  3. 언어 선택

    "한국어" 또는 "English" 중 선택 후 확인.

  4. 설치 경로 선택

    기본값 C:\Program Files\McQrManager\ 권장. 특별한 이유가 없다면 그대로 다음.

  5. 추가 작업 선택

    바탕화면 아이콘 만들기 체크박스 — 필요 시 선택. 기본은 해제 상태.

  6. 설치 진행

    설치 클릭 → 1~2 분 대기. 진행률 바가 100% 에 도달하면 완료.

  7. 완료

    MC QR Code Chip Carrier Manager 실행 체크박스가 선택된 상태로 마침 클릭 → 첫 실행으로 이어집니다.

이 프로그램은 VC++ Redistributable 등 별도 설치 불필요합니다. Tesseract OCR 엔진까지 모두 포함되어 있습니다.

3. 첫 실행

  1. 로그인 다이얼로그

    사용자 이름을 입력하고 확인. 이 이름은 이후 모든 측정 기록에 "작업자"로 저장됩니다.

  2. DB 자동 생성

    최초 실행 시 %LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db 가 자동으로 생성됩니다. 사용자가 따로 할 일은 없습니다.

  3. 메인 윈도우 — 4개 탭 투어

    상단의 4개 탭을 한 번씩 눌러보세요.

    • ATX Mode — 측정 장비의 summary CSV 폴더를 Import 하는 화면
    • Manual Mode — Sweep 화면을 캡처하거나 이미지를 불러와 OCR 로 값 입력
    • CSV Export — ATX / Manual 데이터를 CSV 또는 CSV+Images 로 반출·업로드
    • History — 과거 측정 이력 조회와 Statistics 분석

4. ATX 모드 — 측정 장비 결과 자동 Import

ATX 장비가 생성한 summary_*.csv 파일이 들어있는 폴더를 지정하면 12슬롯을 한 번에 등록합니다.

  1. ATX Mode 진입 → Browse 버튼 클릭
  2. 폴더 선택

    측정 장비가 결과를 저장한 폴더를 선택합니다. 안에 summary_1.csv ~ summary_12.csv 같은 파일들이 있어야 합니다.

  3. 메타 입력

    좌측 입력 패널에서:

    • PO 번호 (필수) — 로트 식별자
    • 주문 수량
    • 프로브 유형 (드롭다운)
    • 생산 일자 (기본: 오늘)
  4. QR 스캐너로 슬롯별 QR 등록

    각 슬롯 카드에 QR 코드 입력 칸이 있습니다. USB QR 스캐너로 순서대로 스캔하면 자동으로 다음 슬롯으로 이동합니다.

  5. 슬롯 값 수정이 필요할 때

    카드를 우클릭하고 수정... 을 선택하면 별도 다이얼로그에서 Probe Type / Frequency / Q / QR ID / Source 를 수정할 수 있습니다. OK 를 누르면 중복 QR 검사를 거쳐 DB 에 자동 저장됩니다.

  6. 저장 또는 업로드
    • 로컬 DB 저장 — Import / QR / 수정 결과는 History 에 반영됩니다.
    • CSV Export 또는 Upload — CSV Export 탭에서 활성 ATX 탭 기준으로 진행합니다.

    둘 다 History 탭에 즉시 반영됩니다.

자주 발생하는 트러블

증상원인 / 해결
"summary CSV 를 찾을 수 없음" 잘못된 폴더 선택. summary_1.csv 등이 있는지 탐색기로 직접 확인.
슬롯 수가 12개가 아님 (예: 11개) 일부 summary 파일 누락. 장비에서 해당 슬롯 재측정.
QR 스캐너로 스캔했는데 반응 없음 QR 입력 칸에 포커스가 있는지 확인. 첫 칸을 마우스로 클릭 후 스캔.

5. Manual Mode — Capture/OCR 자동 입력

Manual Mode 는 Sweep 이미지를 앱 안에서 바로 캡처하고, OCR 로 Frequency / Q-factor 를 읽은 뒤, 방금 생성된 카드에 바로 QR 을 스캔하는 흐름을 기준으로 합니다. 기존처럼 Load 또는 드래그 앤 드롭으로 이미지를 추가하는 방식도 유지됩니다.

  1. Manual Mode 진입 후 Probe Type 탭 선택

    캡처는 현재 선택된 Probe Type 탭에 추가됩니다. Overview 탭에서는 캡처가 시작되지 않습니다.

  2. Zoom-In 이미지 캡처

    📸 Capture 메뉴에서 다음 중 하나를 선택합니다.

    메뉴단축키용도
    Zoom-In · Region CaptureF6드래그한 영역만 캡처
    Zoom-In · Window CaptureF7클릭한 Sweep 창 전체 캡처

    캡처가 끝나면 새 카드가 자동 생성되고 선택됩니다. 좌측 미리보기에는 방금 캡처한 이미지가 표시되며 QR 입력창에 포커스가 이동합니다.

  3. QR 스캐너로 방금 카드에 QR 등록

    QR 을 스캔하면 캡처 파일명이 pending_XXXX.png 에서 slot_XX_{QRID}.png 형식으로 자동 확정됩니다. 파일명을 외울 필요가 없습니다.

  4. Zoom-Out 이미지가 필요한 경우 선택 카드에 추가

    Sweep 프로그램을 Zoom-Out 상태로 바꾼 뒤, 현재 선택된 카드에 Zoom-Out 이미지를 첨부합니다.

    메뉴단축키용도
    Zoom-Out · Region CaptureF8선택 카드의 Zoom-Out 영역 캡처
    Zoom-Out · Window CaptureF9선택 카드의 Zoom-Out 창 전체 캡처
  5. Zoom-In / Zoom-Out 미리보기 전환

    좌측 Zoom-In, Zoom-Out 버튼으로 선택 카드의 이미지를 전환해 확인합니다. Zoom-Out 이미지가 없는 카드에서 Zoom-Out 을 누르면 안내 로그 후 Zoom-In 으로 돌아갑니다.

  6. OCR 결과 확인 + 수기 보정

    OCR 이 끝나면 Frequency / Q-factor 칸에 숫자가 자동 입력됩니다. 값이 틀렸거나 빈 칸이면 직접 수정한 뒤 Apply 를 누릅니다.

  7. 잘못 캡처한 카드 삭제

    카드를 우클릭해 삭제하면 남은 카드가 현재 화면 순서 기준으로 #1, #2, #3... 재정렬됩니다. 앱 내부 캡처 파일도 QR 이 확정된 경우 새 순번에 맞춰 동기화됩니다.

5.1 기존 이미지 Load / Drag & Drop

기존 방식도 그대로 사용할 수 있습니다. Load 버튼으로 파일/폴더를 선택하거나 탐색기에서 JPG / PNG 파일을 카드 영역으로 드래그 앤 드롭합니다.

폴더 선택 시 Zoom-Out backtick 접미사 파일은 슬롯으로 중복 등록되지 않도록 자동 필터링됩니다.

5.2 ROI Calibrator (해상도 다른 이미지 추가 시)

기본 제공 프로파일에 없는 새 해상도의 이미지를 사용할 때 OCR 이 제대로 되지 않으면 상단 🎯 Calibrate OCR 버튼으로 ROI(관심 영역)를 직접 설정합니다.

  1. 🎯 Calibrate OCR 열기
  2. 샘플 이미지 로드

    Load reference image... 버튼으로 문제가 된 해상도의 이미지를 불러옵니다.

  3. ROI 좌표 조정 후 Test OCR

    Frequency ROI 와 Q Factor ROI 를 숫자 영역에 맞춘 뒤 Run OCR on current ROI 로 추출 결과를 확인합니다.

  4. Save

    해당 해상도 기준 ROI 프로파일이 저장되고, 다음부터 같은 해상도 이미지에 자동 적용됩니다.

이미지 미리보기는 원본 파일을 리사이즈하지 않고 현재 화면 영역 안에 맞춰 표시합니다. 캡처 크기나 비율이 달라도 앱 레이아웃은 자동으로 흔들리지 않습니다.

6. History Records — 측정 이력 조회

  1. History 탭 진입 → Records 선택
  2. 필터 설정
    • 날짜 범위 — 캘린더 위젯으로 시작/종료일 선택
    • PO 번호 — 부분 일치 검색
    • Probe Type — 드롭다운
    • Upload Status — pending / uploaded / failed
    • QR 코드 — 특정 QR 로 검색
  3. 정렬

    컬럼 헤더 클릭으로 정렬. 기본은 최신순 (production_date DESC).

  4. 기록 선택 → 슬롯 상세 확인

    기록을 선택하면 하단 상세 패널에서 이미지 미리보기와 슬롯 테이블을 확인할 수 있습니다. History 화면에는 전용 Log 박스가 없으므로 상세 영역을 더 크게 사용할 수 있습니다.

  5. 업로드 실패 건 재전송

    Upload Status = failed 로 필터한 뒤 필요한 이력을 선택해 재전송합니다.

  6. Bundle 백업 / 복원

    선택한 이력은 Export Bundle 로 ZIP 백업할 수 있고, 다른 PC 에서는 Import Bundle 로 복원할 수 있습니다.

History 탭에서는 상단 🎯 Calibrate OCR 버튼이 숨겨집니다. OCR 보정은 ATX / Manual / CSV Export 화면에서 실행하세요.

7. History Statistics — 생산량 분석

7.1 상단 — Today + Period Summary

  • Today Sets — 오늘 측정한 세트(로트) 수
  • Today Slots — 오늘 측정한 슬롯 총 개수
  • Today Complete % — QR 입력 완료율
  • Period Summary — 선택 기간/Probe Type 기준 세트·슬롯·완료율 리스트

7.2 필터

Statistics 상단의 필터:

항목설명
PeriodDaily / Weekly / Monthly 등 집계 단위
Probe Type전체 또는 특정 Probe Type

7.3 차트 해석

영역차트
ProductionProduction Trend, Production by Probe Type
Quality AnalysisFrequency SPC, Q Factor SPC, Frequency Distribution, Q Factor Distribution
그래프는 한 화면에 억지로 모두 끼우기보다 읽을 수 있는 최소 높이를 보장합니다. 필요하면 Statistics 내부 스크롤로 아래 차트를 확인하세요.

8. CSV Export / Bundle — 반출·업로드·백업

데이터 제출용 CSV 와 PC 이전용 Bundle 은 목적이 다릅니다. 고객/서버 제출은 CSV Export, PC 백업/복원은 History Records 의 Export Bundle 을 사용하세요.

8.1 CSV Export 탭

  1. CSV Export 탭 진입

    우측 테이블은 ATX / Manual 2탭으로 나뉩니다. Save CSV, CSV+Images, Upload 는 현재 활성 탭의 데이터 기준으로 동작합니다.

  2. 데이터 완성 상태 확인

    상단 진행률은 활성 Export 탭 기준으로 표시됩니다. 좌측 미리보기는 선택한 슬롯 이미지를 보여줍니다.

  3. Save CSV 또는 CSV+Images 선택
    메뉴결과
    Save CSVCSV 파일만 저장
    CSV + ImagesCSV 와 ZOOMIN/, ZOOMOUT/ 이미지 폴더를 함께 저장
  4. 미완성 데이터 옵션 선택

    QR 미입력 또는 값 누락 슬롯이 있으면 다음 중 선택합니다.

    옵션동작
    QR 있는 값만 반출/업로드QR ID가 있는 슬롯만 CSV 행으로 생성
    전체 슬롯 반출/업로드현재까지 입력된 값을 전체 슬롯 순서대로 CSV 행으로 생성
    취소작업 취소

8.2 CSV 컬럼

순서컬럼명
1QR ID
2생산일자[YYYYMMDD]
3Frequency (KHz)
4Drive (%)
5Q
6Probe Type
Drive (%) 는 현재 화면 입력 항목은 아니지만 서버 업로드 자료 구조 유지를 위해 CSV 에 포함됩니다. 값이 없으면 빈 칸으로 저장됩니다.

8.3 CSV+Images 폴더 구조

생성 폴더명 기본값은 ATX 원본 폴더명 또는 Manual 이미지 공통 부모 폴더명을 우선 사용합니다. 예를 들어 원본 폴더가 P2601002_12M_AC160 이면 기본 Export 폴더명도 동일합니다.

P2601002_12M_AC160/
├── P2601002_12M_AC160_QR.csv
├── ZOOMIN/
└── ZOOMOUT/

8.4 Bundle Export / Import

Bundle 은 History Records 화면에서 사용하는 ZIP 백업/이전 기능입니다.

  1. History Records 에서 Export Bundle

    필요한 기록을 선택하고 Export Bundle 을 클릭합니다. 이미지 포함 옵션을 켜면 Zoom-In 과 Zoom-Out sibling 이미지가 함께 포함됩니다.

  2. 다른 PC 에서 Import Bundle

    Import Bundle 을 클릭하고 ZIP 파일을 선택합니다. 미리보기에서 스키마 버전 / 레코드 수 / 날짜 범위 / 이미지 포함 여부를 확인합니다.

  3. 중복 정책 선택
    정책동작
    Skip (기본)겹치는 QR 이 있으면 해당 세트 건너뜀
    Overwrite기존 세트를 삭제하고 번들 데이터로 교체
    Merge기존 세트는 유지하고 새 QR 슬롯만 추가
Bundle ZIP 은 JSONL + 이미지 파일로 열어볼 수 있으나 수동 편집하지 마세요. 상세 스키마는 docs/export_schema.md 를 참조하세요.

9. 설정

메뉴 → Edit → Settings (또는 Ctrl+,)

주요 항목

  • 사용자 이름 — 로그인 시 입력한 이름 변경
  • 서버 업로드 URL — 회사 내부 서버 엔드포인트
  • OCR ROI 프로파일 — 상단 🎯 Calibrate OCR 버튼으로 Calibrator 진입 (§5.2 참조)
  • 로그 레벨 — All / Warnings+ / Errors only
  • DB 경로 (읽기 전용) — %LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db
  • 캡처 이미지 경로 — %LOCALAPPDATA%\MCQRCodeChipCarrier\captures\
설정은 DB 의 app_settings 테이블에 저장되므로 앱 재시작 후에도 유지됩니다. 업그레이드해도 보존됩니다.

10. 트러블슈팅

Q1. OCR 이 전부 실패해요

  1. 이미지 해상도가 기본 프로파일과 다른지 확인 → ROI Calibrator 에서 신규 프로파일 추가 (§5.2)
  2. 이미지가 흐릿하거나 너무 어두운지 확인 → 카메라 조명·초점 조정
  3. OCR 결과가 항상 None 이면 Tesseract 가 번들에 포함되지 않았을 수 있음 → IT 담당자에게 재설치 요청

Q2. 한글 폴더나 한글 사용자 계정에서 오류가 나요

이 프로그램은 Windows 8.3 단축 경로(short path) 를 자동 변환해 Tesseract 호환 문제를 회피합니다. 재현되면 로그를 첨부해 개발팀에 보고하세요.

Q3. 서버 업로드가 실패해요

  1. Settings 에서 서버 URL 이 올바른지 확인
  2. 네트워크 연결 상태 확인 (사내망 / VPN)
  3. History 에서 "failed" 로 필터 → 우클릭 → Retry Upload

Q4. 앱이 아예 켜지지 않아요

DB 파일 손상 가능성. 데이터가 손실될 수 있으니 신중히:

  1. %LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db 를 다른 이름으로 백업
  2. 원본 DB 삭제
  3. 앱 재실행 (신규 DB 자동 생성)
  4. 백업한 DB 를 SQLite 도구로 열어 복구 가능한 데이터 확인
DB 파일을 삭제하면 모든 이력이 사라집니다. 정기적으로 Export 번들로 백업하세요 (§8).

Q5. SmartScreen 경고가 뜹니다

현재 코드 서명 인증서가 없어 Windows 가 "알 수 없는 게시자" 로 표시합니다. "추가 정보" → "실행" 로 진행하세요. 정상적인 동작입니다.

Q6. 이전 버전(NX 1.0.0) 데이터를 불러오고 싶어요

§12 (부록 — 1.0.0 → 2.0.0 마이그레이션) 을 참조하세요.

Q7. 캡처 이미지가 너무 커서 화면이 깨지는 것 같아요

현재 버전은 원본 이미지를 보존하고 화면에서는 고정 뷰포트 안에 맞춰 표시합니다. 그래도 레이아웃이 어색하면 splitter 를 수동으로 조절하거나 창 크기를 넓혀 확인하세요.

로그 파일 위치

현재는 UI 로그창에 표시됩니다. History 화면은 상세 조회 공간 확보를 위해 전용 Log 박스가 없습니다. 문제 상황은 화면 로그 스크린샷과 함께 지원팀에 보내주세요.

11. FAQ

Q. 여러 사람이 같은 DB 를 공유할 수 있나요?

아니오. 현재는 1 PC = 1 DB 구조입니다. 여러 PC 간 데이터 공유는 Export/Import 번들로 해결합니다.

Q. 데이터는 어디에 저장되나요?

DB 는 %LOCALAPPDATA%\MCQRCodeChipCarrier\chip_carrier.db 에 저장됩니다. Manual 캡처 이미지는 %LOCALAPPDATA%\MCQRCodeChipCarrier\captures\ 아래에 날짜/Probe Type 별로 저장됩니다.

Q. 앱을 업그레이드하면 기존 데이터가 지워지나요?

아니오. DB 는 설치 경로 외부에 있어 업그레이드·재설치 시 보존됩니다.

Q. 인터넷 없어도 쓸 수 있나요?

네. 서버 업로드 기능을 제외한 모든 기능이 오프라인에서 동작합니다.

Q. OCR 이 한국어를 인식하나요?

현재는 영문 + 숫자 전용입니다. 로트카드의 숫자 값 추출에 최적화되어 있습니다. 한국어 OCR 은 차기 버전 검토 항목입니다.

Q. 이 프로그램의 라이선스는?

사내 전용 프로그램입니다. 포함된 Tesseract 는 Apache License 2.0 입니다 (`third_party/tesseract/LICENSE.txt` 참조).

12. 부록 — 1.0.0 (NX) → 2.x (MC) 마이그레이션

이전 버전 NX QR Chip Carrier Manager 1.0.0 사용자가 최신 MC 2.x 로 데이터를 이전하는 절차입니다.

두 버전은 AppId 가 달라 Windows 는 별도 제품으로 인식합니다. 업그레이드 설치가 아니라 신규 설치이며, DB 경로도 변경되므로 자동 이관되지 않습니다.

절차

  1. NX 1.0.0 에서 전체 Export
    • History 탭 → 전체 기간 선택
    • 메뉴 → File → Export Bundle → 이미지 포함 체크
    • 저장 경로: USB / 네트워크 드라이브 / 바탕화면
  2. (선택) NX DB 백업

    %LOCALAPPDATA%\NXQRChipCarrierManager\ 폴더 전체를 복사하여 별도 보관. 만약 번들에 누락이 있어도 SQLite 도구로 복구 가능.

  3. MC 2.x 설치

    §2 절차대로 설치. NX 1.0.0 은 유지해도 되고 언인스톨해도 됩니다.

  4. MC 2.x 에서 Import
    • 메뉴 → File → Import Bundle
    • 1번에서 만든 ZIP 선택
    • 중복 정책: Overwrite (신규 설치라 충돌 없음)
    • Import 클릭 → History 에 모든 데이터 복원 확인
  5. (선택) NX 1.0.0 언인스톨

    제어판 → 앱 및 기능 → NX QR Chip Carrier Manager 제거. DB 폴더(%LOCALAPPDATA%\NXQRChipCarrierManager\)는 수동으로 삭제해야 완전히 정리됩니다.

문제가 생겨도 NX 1.0.0 DB 와 번들 ZIP 이 둘 다 있으면 언제든 복구 가능합니다. 마이그레이션 완료 후 충분한 기간(예: 1개월) 작업 이상 없음을 확인한 뒤 NX 1.0.0 과 구 DB 를 정리하세요.
