# Antigravity Core Execution Rules

1. **Think Before Coding (코딩 전 생각하기)**
   - 코드를 작성하거나 수정하기 전에 반드시 구현 계획과 단계별 Task List를 먼저 세울 것.
   - 상황이 불확실하거나 요구사항이 모호한 경우, 절대 임의로 가정하지 말고 작업을 즉시 멈춘 뒤 나에게 질문할 것.

2. **Simplicity First (단순성 우선)**
   - 내가 명시적으로 요청하지 않은 추가 기능, 과도한 추상화, 불필요한 에러 처리 로직을 임의로 추가하지 말 것.

3. **Surgical Precision (수술과 같은 정밀함)**
   - 외과 수술처럼 지시받은 정확한 목표 지점의 코드만 수정할 것.
   - 현재 구현 목표와 직접적인 관련이 없는 주변 코드를 "개선"한다는 명목하에 임의로 건드리거나 리팩토링하지 말 것.

4. **Goal-Driven Execution (목표 중심의 실행)**
   - 막연한 코드 수정이 아닌, '특정 동작 테스트 통과' 또는 '특정 에러 메시지 해결'과 같은 구체적이고 검증 가능한 목표를 달성하는 데만 집중할 것.

5. **Skill-First Development (스킬 우선 개발)**

   **[세션 시작 시 — 필수]**
   - 새 대화 또는 새 프로젝트에서 코드 작업 요청을 받으면, **첫 번째 행동으로** `.agent/skills/registry.json`을 반드시 읽을 것.
   - registry.json 전체를 읽고 나서야 코드 작업을 시작할 수 있다. 이를 건너뛰는 것은 금지.

   **[작업 시작 전 — 선택적 SKILL.md 읽기]**
   - registry.json의 `tags`와 `source_files`를 기준으로 현재 작업과 관련된 스킬을 **1~3개** 식별할 것.
   - 식별된 스킬의 SKILL.md만 선택적으로 열어 읽을 것. 관련 없는 스킬은 읽지 않아도 된다.
   - 관련 스킬이 없으면 그대로 진행해도 되지만, 그 이유를 명시할 것.

   **[작업 중 — 패턴 일치]**
   - 읽은 스킬의 **Core Patterns**, **Pitfalls & Gotchas** 섹션을 반드시 반영하여 코드를 작성할 것.
   - 스킬에 정의된 패턴과 다르게 구현하려면 반드시 사용자에게 이유를 설명하고 승인을 받을 것.

   **[작업 완료 후 — 검증]**
   - SKILL 16 (16-verification-qa/SKILL.md)의 체크리스트에 따라 결과를 검증할 것.
